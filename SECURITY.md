# Project Protection Checklist

This project is intended to stay private and proprietary.

## Immediate Actions

1. Make the GitHub repository private.
2. Review all collaborators and remove anyone who should not have access.
3. Rotate any secret that may have been committed or shared.
4. Do not store real customer data in the repository.
5. Use local/device storage or a protected backend for sensitive data.

## IP Protection

1. Keep unpublished features and business logic confidential.
2. Use written contractor agreements and NDAs before sharing source code.
3. Register copyright for the software if you want stronger enforcement options.
4. Consider trademark protection for the app name and branding.
5. Talk to an IP attorney before public launch if you think there may be patentable features.

## Repo Hygiene

1. Keep `.env`, local databases, and private notes out of git.
2. Avoid posting full source screenshots publicly.
3. Use branch protection and required reviews if other people will contribute.
4. Back up the repository in a private location you control.
